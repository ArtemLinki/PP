# Что мы создаем

**TechElectro** — интернет-магазин компьютерных комплектующих и электронных компонентов
(микроконтроллеры, сенсоры, питание, робототехника) с встроенным ИИ-чатом, где пользователь
может задать вопрос на естественном языке и получить текстовую консультацию.

---

# Функциональные требования

## Публичная часть (9 экранов)

### Главная страница
- Hero-секция с промпт-полем: пользователь вводит вопрос и переходит в AI-чат (`/ai`)
- Примеры запросов (chips) для быстрого старта
- Фиксированный AI-pill в правом нижнем углу → ведёт в `/ai`
- Адаптивность: отдельный дизайн для desktop и mobile

### Каталог
- Список товаров с пагинацией (до 247 SKU)
- Sidebar с фильтрами: категории (иерархия), диапазон цен, интерфейсы, мощность, бренды
- Поиск по названию/артикулу
- Переключение вид: сетка / список
- Сортировка
- Карточка товара: фото-placeholder, бренд, название, спецификации (mono), цена, кнопка «+ В сборку»

### ИИ-чат (`/ai`)
- Чат-интерфейс с ИИ-помощником
- Пользователь задаёт вопросы о комплектующих (выбор, совместимость, характеристики)
- ИИ отвечает в текстовом формате, история диалога сохраняется (conversationId)
- Поле ввода + отправка, сообщения пользователя и ассистента визуально разделены

### Сборки сообщества
- Каталог BOM-сборок от других инженеров
- Фильтр по категориям (Робототехника, ПК, Умный дом, …)
- Build-карточка: фото, уровень (PRO/MID/EASY), описание, спецификации, автор, рейтинг, цена
- Кнопки «Подробнее» и «Скопировать» (добавить всё в корзину)

### Корзина / Проект
- Корзина позиционируется как «проект» (BOM-лист)
- Позиции: фото, название, бренд/SKU, спецификации, stepper количества, удаление/избранное
- Итоговый блок (sticky): доставка СДЭК, промокод, НДС, кнопка «Оформить заказ»
- Способы оплаты: МИР, VISA, MASTERCARD, СБП
- Сохранение корзины как BOM-сборки

### Проекты пользователя
- Список черновых проектов/сборок
- Статистика: количество проектов, компонентов, общая сумма
- Создание нового проекта, импорт CSV
- Project-строка: статус (active/danger), название, артикул #PRJ-XXX, даты, теги, итог
- Контекстное меню: переименовать, превратить в сборку, дублировать, удалить
- Недоступность кнопки «Оформить» если есть позиции не в наличии

### Аутентификация
- Вход: email/пароль + OAuth (Google, GitHub)
- Регистрация: два типа аккаунта — B2C (частный) и B2B (юр. лицо / ИП)
- Поля: ФИО, email, пароль, подтверждение пароля, телефон (опционально)
- Чекбокс согласия с условиями

## Административная панель (6 экранов)

### Дашборд
- KPI: количество товаров, заказы за месяц, выручка, количество AI-запросов
- График продаж по дням (SVG line-chart)
- Популярные категории (горизонтальные бары)
- Алерт о заканчивающихся SKU
- Последние заказы + топ товаров

### Управление товарами
- Таблица всех SKU с поиском, фильтрами (категория, бренд, цена, статус)
- Фильтр: Все / Опубликованы / Черновики / Архив / Заканчиваются
- Действия на строке: редактировать, дублировать, удалить
- Массовый выбор через чекбоксы

### Модалка добавления / редактирования товара
- Двухколоночный layout (основные поля + aside с настройками публикации)
- Поля: название, категория, бренд, SKU, цена, изображения (до 8), спецификации (key/value), описание
- Aside: статус публикации, остаток, старая цена, теги
- Автосохранение черновика

### Управление категориями
- Иерархическое дерево (2 уровня)
- Добавление новой категории через inline input
- Toggle видимости (switch 28×18)
- Статистика: всего / корневых / макс. глубина / скрытых

### Управление брендами
- Сетка карточек производителей: лого (текст-аббревиатура), название, URL/страна
- Статистика: SKU, выручка, процент в наличии

### Управление заказами
- Таблица заказов с фильтрами по статусу
- Статистика: всего / требуют внимания / оплачено / выручка за 30д
- Строка заказа: номер, покупатель (аватар+имя+email), позиций, сумма, статус, дата

# Технологические требования

- Полная адаптивность: mobile (`< 768px`) — нижний таб-бар, desktop — топ-навигация
- Тёмная тема по умолчанию (фирменный стиль TechElectro)
- Pixel-perfect соответствие дизайн-мокапам из `design_handoff_techelectro/mockups/`
- Все цены — в копейках в БД, форматирование через `formatPrice()`
- Авторизация через JWT, HttpClient автоматически добавляет `Authorization: Bearer`
- Система мок-сервисов для разработки без бэкенда (`NEXT_PUBLIC_USE_MOCKS=true`)
- Все формы с валидацией на клиенте
- Общие TypeScript DTO в `common/` — единый источник правды для frontend и backend

# Архитектура

## Структура монорепо

```
ecommerce/
├── common/
│   └── dto/                 # Общие TypeScript-интерфейсы (без фреймворк-зависимостей)
│       ├── product.dto.ts
│       ├── category.dto.ts
│       ├── cart.dto.ts
│       ├── order.dto.ts
│       ├── ai.dto.ts
│       ├── user.dto.ts
│       └── common.dto.ts    # ApiResponse<T>, Paginated<T>, PriceDto, ID
├── frontend/                # Next.js приложение
└── backend/                 # NestJS приложение
```

## Слои Frontend

```
frontend/src/
├── app/                     # Страницы (роутинг)
│   ├── (public)/            # Публичные маршруты
│   └── admin/               # Защищённая панель администратора
├── components/              # UI-компоненты
│   ├── layout/              # AppShell, Header, MobileBottomNav
│   ├── product/             # ProductCard, ProductGrid
│   ├── ai/                  # AiAssistantPill, AiChatWindow
│   └── admin/               # AdminSidebar, KPICard, DataTable
├── lib/
│   ├── api/                 # HttpClient, endpoints, config
│   ├── services/            # *ApiService / *MockService per domain
│   ├── store/               # Zustand-сторы (cart, auth, ui, ai)
│   └── mocks/               # Локальные данные для мок-сервисов
└── theme/                   # Mantine-тема с токенами TechElectro
```

## Слои Backend (NestJS)

```
backend/src/
├── modules/
│   ├── auth/                # JWT, регистрация, login, OAuth
│   ├── users/               # Профиль, B2C/B2B
│   ├── products/            # CRUD + поиск + фильтрация
│   ├── categories/          # Иерархия
│   ├── brands/              # Производители
│   ├── cart/                # Корзина per user
│   ├── orders/              # Заказы, статусы
│   └── ai/                  # Gemini API чат, хранение диалогов
├── prisma/                  # Schema, миграции
└── common/                  # Guards, filters, interceptors, pipes
```

## Поток данных

```
Пользователь → Next.js page → useQuery(services.X.method())
                                    ↓
             NEXT_PUBLIC_USE_MOCKS=true  →  *MockService (локальные данные)
             NEXT_PUBLIC_USE_MOCKS=false →  *ApiService → HttpClient → NestJS API → PostgreSQL

Чат: Frontend useAiStore.send() → POST /ai/prompt → NestJS AiService → Gemini API → ответ
```
