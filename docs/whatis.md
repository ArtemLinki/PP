# Что мы создаем

**TechElectro** — интернет-магазин компьютерных комплектующих и электронных компонентов
(микроконтроллеры, сенсоры, питание, робототехника) с двухрежимным поиском: обычный нечёткий
поиск по каталогу и ИИ-помощник, который умеет не только отвечать текстом, но и выполнять
действия на сайте (подбирать товары, добавлять в корзину).

---

# Функциональные требования

## Публичная часть

### Главная страница — двухрежимный инпут

Центральный элемент страницы — инпут с кнопкой **AI** рядом.

**Режим 1 — обычный поиск** (по умолчанию, кнопка AI не нажата):
- Нечёткий полнотекстовый поиск по названию, описанию и тегам товаров
- При отправке → переход на `/catalog?q=...` с результатами

**Режим 2 — AI** (кнопка AI нажата / активна):
- Тот же инпут, но запрос уходит в AI-чат
- Переход на `/ai` с начальным сообщением, или инлайн-ответ на той же странице
- ИИ может ответить текстом и/или показать блок «Подобрано для вас» с карточками товаров

Дополнительно на главной:
- Chips с примерами запросов (для обоих режимов)
- Фиксированный AI-pill в правом нижнем углу → ведёт в `/ai`
- Адаптивность: отдельный дизайн для desktop и mobile

### Каталог

- Список товаров с пагинацией
- Поиск по названию/артикулу (тот же нечёткий поиск, что и на главной)
- **Базовые фильтры** (только): диапазон цены, выбор категории, выбор бренда
- Переключение вида: сетка / список
- Сортировка (по цене, новизне, популярности)
- Карточка товара: фото, бренд, название, спецификации (mono), цена, кнопка «+ В корзину»

### AI-чат (`/ai`)

- Чат-интерфейс с ИИ-помощником
- Пользователь задаёт вопросы: выбор комплектующих, совместимость, характеристики, просьба «собери мне ПК за 80 000₽»
- **Текстовый ответ**: ИИ объясняет, советует, сравнивает
- **Блок рекомендаций** (если ИИ выполнил function call `searchProducts`): карточки-плашки «Подобрано для вас» с фото, названием, ценой и кнопкой перехода к товару
- **Добавление в корзину из чата**: ИИ может выполнить `addToCart` — пользователь видит уведомление «Добавлено в корзину»
- История диалога сохраняется (conversationId), загружается при возврате
- Индикатор загрузки пока ИИ отвечает или выполняет действие

### Корзина

- Позиции: фото, название, бренд/SKU, спецификации, stepper количества, удаление
- Итоговый блок (sticky): доставка, промокод, НДС, кнопка «Оформить заказ»
- Способы оплаты: МИР, VISA, MASTERCARD, СБП

### Аутентификация

- Вход: email/пароль + OAuth (Google, GitHub)
- Регистрация: два типа аккаунта — B2C (частный) и B2B (юр. лицо / ИП)
- Поля: ФИО, email, пароль, подтверждение пароля, телефон (опционально)
- Чекбокс согласия с условиями

## Административная панель

### Дашборд
- KPI: количество товаров, заказы за месяц, выручка, количество AI-запросов
- График продаж по дням (SVG line-chart)
- Популярные категории (горизонтальные бары)
- Алерт о заканчивающихся SKU
- Последние заказы + топ товаров

### Управление товарами
- Таблица всех SKU с поиском, фильтрами (категория, бренд, цена, статус)
- Фильтр: Все / Опубликованы / Черновики / Архив / Заканчиваются
- Действия: редактировать, дублировать, удалить
- Массовый выбор через чекбоксы

### Модалка добавления / редактирования товара
- Двухколоночный layout (основные поля + aside с настройками публикации)
- Поля: название, категория, бренд, SKU, цена, изображения (загрузка в MinIO, до 8 штук), спецификации (key/value), описание
- Aside: статус публикации, остаток, старая цена, теги
- Автосохранение черновика

### Управление категориями
- Иерархическое дерево (2 уровня)
- Добавление новой категории через inline input
- Toggle видимости
- Статистика: всего / корневых / макс. глубина / скрытых

### Управление брендами
- Сетка карточек производителей: лого (загружаемое или текст-аббревиатура), название, URL/страна
- Статистика: SKU, выручка, процент в наличии

### Управление заказами
- Таблица заказов с фильтрами по статусу
- Статистика: всего / требуют внимания / оплачено / выручка за 30д
- Строка заказа: номер, покупатель, позиций, сумма, статус, дата

# Технологические требования

- Полная адаптивность: mobile (`< 768px`) — нижний таб-бар, desktop — топ-навигация
- Тёмная тема по умолчанию
- Pixel-perfect соответствие дизайн-мокапам из `design_handoff_techelectro/mockups/`
- Все цены — в копейках в БД, форматирование через `formatPrice()`
- Авторизация через JWT, HttpClient автоматически добавляет `Authorization: Bearer`
- Система мок-сервисов для разработки без бэкенда (`NEXT_PUBLIC_USE_MOCKS=true`)
- Все формы с валидацией на клиенте
- Общие TypeScript DTO в `common/` — единый источник правды для frontend и backend
- Файловое хранилище: MinIO (S3-compatible, self-hosted)
- Нечёткий поиск: PostgreSQL `pg_trgm`

## AI DTO — ключевые типы

```typescript
// Запрос в чат
interface AiChatRequestDto {
  message: string;
  conversationId?: ID;
}

// Ответ из чата
interface AiChatResponseDto {
  conversationId: ID;
  reply: AiMessageDto;
  toolResults?: AiToolResultDto;  // присутствует если ИИ вызвал инструменты
}

// Результаты выполненных инструментов
interface AiToolResultDto {
  recommendedProducts?: ProductDto[];   // searchProducts → показать карточки
  addedToCart?: CartDto;               // addToCart → обновить корзину
}
```

## Архитектура AI Function Calling

```
Frontend (useAiStore.send)
    │
    ▼
POST /ai/chat  {message, conversationId?}
    │
    ▼
AiService (NestJS)
    ├── Загружает историю диалога из БД
    ├── Формирует системный промпт (роль консультанта TechElectro)
    ├── Передаёт в Gemini: history + message + tools (searchProducts, addToCart, getProductDetails)
    │
    ▼
Gemini API
    ├── Вариант A: возвращает текстовый ответ → AiService сохраняет, возвращает клиенту
    └── Вариант B: вызывает tool(s) → AiService → AiExecutor выполняет реальный сервис
            │
            ▼
        Результат tool передаётся обратно в Gemini → финальный текст
            │
            ▼
        AiService сохраняет сообщение + tool results, возвращает клиенту:
        { reply: AiMessageDto, toolResults: { recommendedProducts?, addedToCart? } }
    │
    ▼
Frontend
    ├── Показывает текстовый ответ
    └── Если toolResults.recommendedProducts → рендерит блок «Подобрано для вас»
        Если toolResults.addedToCart → обновляет useCartStore, показывает уведомление
```

## Поток данных (основной)

```
Пользователь → Next.js page → useQuery(services.X.method())
                                    ↓
             NEXT_PUBLIC_USE_MOCKS=true  →  *MockService (локальные данные)
             NEXT_PUBLIC_USE_MOCKS=false →  *ApiService → HttpClient → NestJS API → PostgreSQL

Файлы: NestJS FilesModule → MinIO (S3-compatible)
```
